# 前端设计规范

本规范来自 `D:\code\ai-hot-monitor\AGENTS.md` 的前端设计规则，并结合 Gewu Agent 的产品定位做了项目化整理。后续每次设计或实现前端页面时，默认遵循本文件，不需要额外提示。

## 1. 设计角色

Gewu Agent 的前端设计应避免千篇一律的管理后台模板。它是学习者自己的智能体工作台，既要专业、清晰、可长期使用，也要有一点反主流的个性。

目标气质：

- 专业但不冷冰冰。
- 有信息密度，但不拥挤。
- 有视觉记忆点，但不喧宾夺主。
- 像一个长期陪伴学习和调试智能体的工作台，而不是营销落地页。

## 2. 绝对禁止项

### 2.1 配色禁止

- 禁止使用紫色、靛蓝色、蓝紫渐变作为主视觉。
- 避免 `#6366F1`、`#8B5CF6` 这类常见 AI SaaS 默认色。
- 禁止直接使用 Tailwind 默认色板作为最终视觉系统。
- 禁止大面积纯平背景色；背景需要有轻微噪点、纹理、分层色块或细腻渐变。

### 2.2 布局禁止

- 禁止 Hero + 三卡片的模板化首页。
- 禁止所有内容完美居中排布。
- 禁止等宽多栏机械布局；需要通过侧栏、轨迹栏、引用栏、调试区形成有意义的不对称。
- 禁止为了好看牺牲信息扫描效率。

### 2.3 文案禁止

- 禁止 Lorem Ipsum。
- 禁止空泛的专业名词堆叠。
- 禁止无意义营销话术。
- 避免被动语态和过长句子。

### 2.4 组件禁止

- 禁止直接暴露 Shadcn、Material UI、Element Plus 等组件库默认观感。
- 禁止使用 emoji 作为功能图标。
- 禁止线性、生硬、无缓动设计的动效。
- 禁止卡片套卡片和过度装饰。

## 3. 必须遵守项

### 3.1 文案风格

- 口语化，像朋友聊天。
- 具体化，有数字、有场景、有对象。
- 可以轻微幽默、自嘲或带一点锋芒。
- 页面短文案尽量每句不超过 15 个字。
- 按钮文案用明确动作，例如“开始入库”“重建索引”“查看引用”“继续追问”。

### 3.2 图片与图标系统

- 图标默认使用 Iconify 图标库：https://iconify.design
- 占位图可使用 Picsum Photos：https://picsum.photos
- 真实图片可使用 Pexels 搜索：https://www.pexels.com
- 插画可使用 unDraw：https://undraw.co
- 功能按钮优先使用图标，复杂或不常见图标需要 tooltip。
- 图标风格要统一，不混用多套视觉语言。

### 3.3 Gewu Agent 专属界面原则

- 智能体工作台优先采用三栏或非对称布局：智能体/知识库选择、聊天主区、引用与工具轨迹。
- 知识库页面要强调文件状态、切片数量、索引状态、失败原因和重建入口。
- RAG 调试页面要能快速看到 query、TopK、score、chunk、来源和最终上下文。
- LeManus 调试台要突出 Reason、Act、Observe、Reflect 的执行轨迹。
- 阶段 0 模型配置页面要清楚区分 DeepSeek v4 flash 和 DeepSeek v4 pro；Ollama 后续再出现。

## 4. 色彩方向

推荐方向：

- 墨黑、炭灰、纸白、苔绿、铜色、暖红、雾蓝等更有质感的组合。
- 可以使用细腻纹理、轻微噪点、局部分层色块。
- 重点色只用于行动按钮、状态、当前选中项和关键指标。

避免方向：

- 默认 AI 紫。
- 大面积蓝紫渐变。
- 全页面单一蓝灰。
- 过度玻璃拟态。
- 过度奶油色、咖啡色或单一暖色。

## 5. 交互与动效

- 动效要服务状态变化，例如入库进度、检索展开、工具调用轨迹、消息流式输出。
- 默认使用自然缓动，不使用生硬线性动画。
- 页面切换和折叠展开要轻，不打断阅读。
- 重要操作需要明确状态：进行中、成功、失败、可重试。

## 6. 组件定制要求

- Button：必须有明确层级，主要动作、次要动作、危险动作区分清楚。
- Card：只用于真实的信息单元，不用于把整个页面包起来。
- Table/List：优先保证扫描效率，状态、时间、来源、操作要清晰。
- Dialog：用于确认、配置和调试，不承载大段说明文字。
- Sidebar：承载智能体、知识库、会话等长期导航。
- Right Panel：承载引用来源、工具轨迹、检索片段和运行日志。

## 7. 页面验收标准

每个前端页面完成前，需要自查：

- 是否一眼能看出当前页面在做什么？
- 是否避免了模板化 Hero + 卡片？
- 是否没有使用紫色/靛蓝/蓝紫渐变主视觉？
- 是否没有直接暴露组件库默认风格？
- 是否能清楚展示状态、来源、轨迹或下一步动作？
- 是否所有图标都有统一来源和语义？
- 是否移动端和桌面端都没有文字溢出或重叠？
