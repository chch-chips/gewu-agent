# 路线图与里程碑

## 阶段 0：项目初始化

目标：让项目可启动、可开发、可演示。

任务：

- 初始化 Spring Boot 后端项目。
- 初始化 Vue + TypeScript 前端项目。
- 建立基础目录结构和代码规范。
- 接入 DeepSeek 云模型。
- 实现普通聊天接口和流式响应。

验收：

- 前端可选择模型并发送消息。
- 后端可以返回流式回答。
- README 中有启动命令。

## 阶段 1：PDF 本地知识库

目标：完成 RAG 最小闭环。

任务：

- 创建知识库。
- 上传 PDF。
- 解析 PDF 文本。
- 文本切分。
- 生成 embedding。
- 写入向量数据库。
- 用户提问时召回 chunk。
- 生成带引用的回答。

验收：

- 面试鸭 PDF 可以入库。
- 用户询问知识点时，回答能展示来源文件和页码。
- 提供检索调试接口，能看到 TopK 片段。

## 阶段 2：面试助手产品化

目标：把 RAG 问答变成真正的面试学习产品。

任务：

- 智能体配置：提示词、模型、知识库。
- 回答模式：标准答案、面试口述、追问模式。
- 会话历史。
- 薄弱点总结。
- 固定评测题集。

验收：

- 可以进行一次完整模拟面试。
- 能生成面试复盘。
- 能比较不同模型/检索参数对回答质量的影响。

## 阶段 3：408 考研学习助手

目标：支持课程资料型知识库。

任务：

- 按课程、章节、标签管理文档。
- 支持章节摘要。
- 支持考点/易错点/题型角度回答。
- 支持 PPT/PPTX 文本抽取。

验收：

- 408 课程 PDF 可以按课程维度检索。
- PPT 课件可抽取文字并参与问答。
- 能输出一份章节复习清单。

## 阶段 4：LeManus ReAct 智能体

目标：自研一个可解释的 Agent Runtime。

任务：

- 定义 AgentRun / AgentStep。
- 实现 Reason / Act / Observe / Reflect 循环。
- 实现工具注册表。
- 接入知识库检索工具。
- 接入 MCP 工具。
- 增加最大步数、超时、工具白名单。

验收：

- LeManus 能完成至少 3 类多步任务。
- 前端能展示执行轨迹。
- 工具调用失败时能中止或修正。

## 阶段 5：评测与工程亮点

目标：让项目从“能跑”升级到“有工程深度”。

任务：

- 建立 RAG 评测集。
- 指标：召回命中、引用准确性、回答完整性、幻觉率。
- 支持不同切分策略对比。
- 支持不同模型/向量库对比。
- 增加日志、监控、token 统计。

验收：

- 有一组可复现实验结果。
- 简历中可以量化描述项目效果。
- 面试时可以讲清架构、难点和权衡。

## 推荐开发顺序

1. 普通聊天。
2. PDF 入库。
3. RAG 问答。
4. 面试助手。
5. 知识库管理页面。
6. PPT/PPTX 支持。
7. LeManus 最小 ReAct 循环。
8. MCP 接入。
9. 评测中心。
10. 简历级优化。

## 风险清单

| 风险 | 表现 | 处理 |
| --- | --- | --- |
| Spring Boot 4 + Spring AI 2.0 GA 兼容问题 | 依赖冲突、API 变动 | 使用 BOM，以 2.0 GA Reference 为准，避免 Snapshot API |
| RAG 回答效果不稳定 | 召回片段不准、答案泛化 | 做检索调试和评测集 |
| PPT 解析质量一般 | 图表、公式、图片丢失 | 首期只承诺文本和备注，后续 OCR/多模态 |
| Agent 容易失控 | 循环调用、工具误用 | 最大步数、白名单、超时、执行日志 |
| 项目范围过大 | 长期做不完 | 严格按阶段验收，先闭环再增强 |
